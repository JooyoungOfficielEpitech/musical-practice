## 3. 불변식 (깨지면 버그)

### 싱크 3중 방어
1. **서버**: 모든 duration은 음표 기호(type×dots×tuplet)에서 역산. 마디 합 = 박자표 정확히. 초과는 마디선에서 절단, 부족은 쉼표 패딩. **스케일 압축 영구 금지.**
2. **앱 파서**: 모든 파트를 공유 바 그리드에 마디 단위 앵커링. 파트별 누적 오차 전파 불가능.
3. **플레이어**: notes 배열 교체(파트 토글) 시 인덱스 리셋 + 현재 위치 재스케줄. 타이머 콜백은 ref 경유 (스테일 클로저 금지).

### 보이는 것 = 들리는 것
화면의 음표 기호와 재생되는 길이는 항상 일치한다. 위반 = 서버 버그.

### 템포
- 파서가 읽은 템포는 **[40, 200] BPM 범위 밖이면 불신** → 기본값 100.
- 사용자가 조절한 템포 배율은 **악보별로 영구 저장**, 다시 열면 그대로.
- OMR의 템포 마킹 오독은 전제 조건: 사용자가 한 번 맞추면 끝나야 한다.

### 데이터 신선도
- 악보를 열 때마다 서버 최신 결과를 재다운로드 (오프라인이면 캐시).
- 서버 재처리가 폰에 자동 반영되어야 한다 (stale 캐시 금지).

### 오디오 UX
- 정지/일시정지는 즉각적 (마스터 버스 절단).
- 백그라운드 복귀 시 밀린 노트 버스트 금지.
- 커서는 오디오 클록에서 파생 (별도 타이머 금지).

## 4. 아키텍처 고정 사항

- **백엔드**: Supabase만 (큐 `omr_jobs`, 버킷 `omr-pdfs`/`omr-results`). 커스텀 API 서버 금지.
- **OMR**: 로컬 Mac의 파이프라인 (homr) — cmux 워크스페이스에서 상주, 5초 폴링.
- **Supabase 무료 티어**: keep-alive 크론(GitHub Actions, main 브랜치) 필수 유지.
- **배포**: JS 변경 = `eas update`(OTA, 즉시). 네이티브 변경만 EAS 로컬 빌드 → TestFlight.
- **테스트**: TDD. 앱 jest + 서버 pytest 전부 그린이 머지 조건.

## 5. 디자인 ("Sheet & Ink")

- 라이트: 따뜻한 종이(#FAF9F7) + 잉크 CTA(#1C1917). 다크: 트루블랙(#0C0A09) + 앰버(#F59E0B).
- 버튼은 반드시 `colors.primary` + `colors.buttonText` 페어 (모드 간 잉크↔종이 반전).
- 재생 커서 = 앰버.
